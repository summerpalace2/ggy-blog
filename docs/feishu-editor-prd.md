# 飞书式写作编辑器 PRD

- 版本：`v1.0`
- 状态：`Draft`
- 适用范围：`src/app/write/page.tsx` 对应的写作编辑器

## 1. 背景

当前博客项目的写作页已经具备“能写、能存、能发”的基础能力，但交互方式更接近传统富文本编辑页，仍然缺少飞书云文档那种“块级写作”的心流体验。

本 PRD 的目标不是直接复刻飞书全部功能，而是把写作页升级为一个**块驱动、光标附近完成操作、结构与文本分离**的编辑器。

## 2. 产品目标

### 2.1 业务目标
- 降低写文章的结构操作成本
- 让标题、正文、图片、表格、代码块等内容以统一方式组织
- 让文章天然具备目录、层级和结构感
- 支持后续扩展评论、修订、协作等能力

### 2.2 用户目标
- 写作时尽量不离开正文区域
- 通过 `+`、`/`、快捷键完成大多数插入和转换操作
- 支持图片、代码、表格、引用、待办等常用块
- 保存后可直接发布为博客文章

## 3. 设计原则

1. **块即结构**：段落、标题、图片、代码块、表格都应视为独立 Block。
2. **命令统一**：`+` 和 `/` 只是两种入口，背后使用同一套块命令。
3. **操作靠近光标**：插入、转换、格式化尽量在当前输入位置完成。
4. **默认连续写作**：Enter 应尽量延续写作流，而不是打断。
5. **可扩展优先**：数据结构和组件边界要为未来协作、评论、修订预留空间。

## 4. 用户画像

| 角色 | 场景 | 核心诉求 |
|---|---|---|
| 内容作者 | 写技术文章、周报、博客 | 快速插入结构、保持专注 |
| 管理员 | 发布、维护、更新文章 | 保存稳定、格式可控 |
| 协作者（未来） | 评论、建议、修订 | 定位准确、上下文清晰 |

## 5. 页面结构

### 5.1 页面层级
- 顶部栏：返回、标题、发布、更多操作、保存状态
- 主内容区：标题区 + 正文块列表
- 左侧辅助区：目录、大纲、块级悬停入口
- 右侧辅助区：评论、版本、设置等未来能力

### 5.2 正文层级
- 文档标题
- 封面图（可选）
- 文档元信息：分类、标签、摘要
- 正文 Block 列表
- 底部状态区：块数、字数、保存状态

## 6. 核心交互

### 6.1 `+` 插入按钮

**触发条件**
- 鼠标悬停到空段落左侧
- 鼠标悬停到块之间的空位
- 光标进入空块时自动显现

**按钮作用**
- 插入新的 Block
- 提供鼠标入口，降低命令学习成本
- 与 `/` 菜单共用同一命令源

**执行逻辑**
1. 识别当前块上下文
2. 显示块类型面板
3. 用户选择块类型
4. 当前块原地转换或在当前位置插入新块
5. 新块自动聚焦，继续写作

### 6.2 `/` 命令菜单

**触发条件**
- 在正文块中输入 `/`
- 在空块中按 `/`
- 通过快捷键或菜单唤起同一命令面板

**按钮作用**
- 快速插入块
- 搜索命令
- 按键盘选择命令项

**执行逻辑**
1. 捕获 `/` 输入
2. 打开命令面板
3. 根据输入内容过滤命令项
4. 回车确认或鼠标点击确认
5. 将结果应用到当前块上下文
6. 关闭菜单并保持光标连贯

### 6.3 块类型切换

**触发方式**
- 左侧块类型标签点击
- `+` 菜单选择类型
- `/` 菜单选择类型
- 快捷键切换

**执行逻辑**
- 当前内容保留，块语义发生变化
- 需要清理旧类型的前缀标记
- 例如标题前缀、列表前缀、引用符号等应被统一处理

### 6.4 选区格式工具条

**触发条件**
- 用户选中正文中的一段文本

**支持动作**
- 加粗
- 斜体
- 下划线
- 删除线
- 文字颜色
- 背景高亮
- 插入链接

**执行逻辑**
- 只改变 inline 样式，不改变 block 类型
- 工具条在选区附近出现
- 点击工具条后保持当前编辑上下文

### 6.5 键盘交互

- `Enter`：普通段落中拆分块；列表/待办中生成下一项；代码块中换行
- `Shift+Enter`：同块软换行
- `Backspace`：空块删除；块头退格时尝试与上一块合并
- `Delete`：块尾删除时尝试与下一块合并
- `Tab`：在可切换场景循环块类型或缩进级别
- `Esc`：关闭菜单与浮层
- `Ctrl/Cmd+S`：保存或发布

## 7. 功能需求清单

### 7.1 文档基础能力
- 标题独立编辑
- 摘要输入
- 分类选择：`tech` / `life`
- 标签输入
- 封面图设置
- 保存草稿
- 发布文章

### 7.2 块编辑能力
- 段落块
- 一级/二级/三级标题块
- 引用块
- 无序列表
- 有序列表
- 待办清单
- 代码块
- 图片块
- 表格块
- 分割线
- 提示框
- 折叠块
- 数学公式
- 嵌入内容

### 7.3 内容辅助能力
- 目录自动生成

## 8. 块类型细则

| 类型 | 点击/输入后的行为 | 特别说明 |
|---|---|---|
| Paragraph | 默认正文块 | 最常见的写作块 |
| H1 | 升级为一级标题 | 通常用于文章主标题 |
| H2 | 升级为二级标题 | 文章章节标题 |
| H3 | 升级为三级标题 | 小节标题 |
| Quote | 转为引用块 | 左侧蓝线、浅色背景 |
| List | 转为无序列表 | Enter 继续下一项 |
| Ordered List | 转为有序列表 | 自动编号 |
| Todo | 转为待办清单 | 带复选框 |
| Code | 转为代码块 | 等宽字体、语法高亮 |
| Image | 转为图片块 | 支持上传/粘贴/URL |
| Table | 转为表格块 | 支持行列编辑 |
| Divider | 插入分割线 | 只做结构切分 |
| Callout | 转为提示框 | 可切换样式变体 |
| Toggle | 转为折叠块 | 支持展开/收起 |
| Formula | 转为公式块 | 支持 LaTeX |
| Embed | 转为嵌入块 | 支持网页或卡片 |

## 9. 内容发布流程

1. 校验标题是否为空
2. 序列化 block 结构
3. 转换为发布格式（Markdown / MDX / HTML）
4. 调用后端保存接口
5. 保存成功后跳转文章页
6. 清理草稿缓存

## 10. 非功能要求

- **性能**：输入过程中不应明显卡顿
- **稳定性**：内容不得因菜单、切换、拖拽而丢失
- **可扩展性**：新增 Block 类型时不应改动整页逻辑
- **可维护性**：命令、渲染、状态、弹窗要解耦
- **可访问性**：支持键盘完成主要操作

## 11. 验收标准

- 可以通过 `+` 插入常用 Block
- 可以通过 `/` 搜索并插入同一套 Block
- 可以通过键盘完成写作、拆分、删除和类型切换
- 可以插入并编辑图片、代码和表格
- 可以自动恢复草稿
- 可以稳定发布文章

## 12. 迭代优先级

### P0
- 块编辑主流程
- `+` / `/` 统一命令系统
- 标题、正文、引用、列表、代码、图片
- 保存、发布、草稿恢复

### P1
- 表格、公式、折叠块、提示框、嵌入
- 目录联动
- 选区格式工具条
- 更完整的快捷键

### P2
- 评论
- 修订
- 协作
- 历史版本
- 分享与权限
