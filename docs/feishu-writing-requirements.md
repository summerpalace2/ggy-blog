# 飞书云文档写作能力需求分析

## 1. 文档目标

本文档用于对标飞书云文档的“写文章”体验，沉淀成我们网站写文章模块的产品需求与执行流程。

本次分析基于两类信息：

- 本地项目现有实现，主要是 [`src/app/write/page.tsx`](/D:/agent/ggy-blog/src/app/write/page.tsx)
- 飞书云文档公开可编辑页面的实际体验结果

> 说明：本次对飞书的体验重点放在“写作/编辑”链路，不是完整的知识库管理或权限管理评估。

## 2. 飞书云文档写作页的观察结论

### 2.1 页面结构

飞书文档的页面结构不是单一文本框，而是典型的“文档级页面 + 块级内容编辑”模式。

已观察到的页面元素：

- 顶部文档操作区：分享、编辑模式切换、更多入口
- 左侧目录/大纲区：根据标题自动生成锚点导航
- 文档主体区：标题、正文、表格、引用等内容以块形式呈现
- 评论入口：正文下方可以看到评论数量和跳转首条评论
- 封面与图标入口：页面顶部可添加图标、封面
- 编辑模式入口：编辑 / 修订 / 阅读三种模式切换

### 2.2 可编辑结构

在编辑态中，文档主体并非一个大输入框，而是多个 `contenteditable` 区块的组合：

- 页面根块
- 标题块
- 正文块
- 空白占位块
- 后续内容块

这意味着飞书的写作本质是“块编辑器”，而不是传统富文本 textarea。

### 2.3 真实可见的写作引导

在正文区域可以看到明确提示：

- `输入 "/" 快速插入内容`

这说明飞书的块插入是通过命令菜单驱动的，而不是依赖单纯工具栏按钮。

### 2.4 当前可确认的编辑能力

根据页面可见结构与编辑态观察，飞书至少具备以下能力：

- 标题编辑
- 正文块编辑
- 大纲自动生成
- 封面图 / 图标设置
- 评论查看与跳转
- 编辑 / 修订 / 阅读模式切换
- 斜杠菜单插入块
- 基础块类型支持
- 表格内容展示与编辑

### 2.5 需要继续补测但已高度可疑的能力

以下能力在界面中强烈暗示存在，但本轮没有逐项做完整交互闭环验证：

- 块级拖拽排序
- 选中文本浮动格式工具条
- 块级浮动工具条：段落 / 标题 / 图片 / 表格左侧悬停时出现各自工具条
- 链接插入与编辑
- 图片上传 / 粘贴 / 拖拽
- 表格插入与行列操作
- 折叠内容 / 多级标题折叠
- 引用、待办、列表等块的快捷转换
- 版本修订 / 建议模式的具体行为

## 3. 飞书写作的执行逻辑

### 3.1 总体逻辑

飞书写作可以概括为下面的执行链路：

1. 用户进入文档
2. 选择编辑模式
3. 在块级内容中输入文字
4. 通过 ` / `、快捷键、工具栏或菜单触发块类型变化
5. 文档自动生成目录与层级结构
6. 用户可随时插入封面、图片、表格、链接、评论
7. 保存状态由文档系统自动维护，用户感知为实时保存/协作

### 3.2 块级执行逻辑

飞书的内容编辑不是“全文统一富文本”，而是“每个块独立处理”。

典型执行方式：

- 标题块负责文档主题
- 正文块负责连续叙述
- 引用块、列表块、待办块、表格块等具有自己的编辑语义
- 空白块可作为新块插入点
- 目录块由标题自动派生，不手工维护

### 3.3 交互触发方式

飞书写作中的常见触发方式包括：

- 直接输入
- `/` 命令菜单
- 选中文本后出现格式浮层
- 鼠标悬停块左侧边缘触发块级浮动工具条
- 点击顶部或侧边的文档功能入口
- 键盘快捷键

### 3.4 重要产品特征

飞书的写作体验核心不是“功能多”，而是“低干扰、高连续性”：

- 用户不必频繁离开正文
- 大部分结构操作可在当前光标附近完成
- 标题和目录自动联动
- 编辑行为具备明显的块上下文

## 5. 需求定义

### 5.1 写作页面目标

写作页应对标飞书云文档的“低打扰块编辑”体验，让用户可以像写文档一样完成：

- 写标题
- 写正文
- 插入块
- 调整结构
- 插入图片/表格/引用/待办/公式
- 预览并发布文章

### 5.2 核心需求

#### 5.2.1 文档级结构

- 页面采用文档级布局，不使用单一大文本框
- 标题、封面、正文、目录、辅助操作区分层展示
- 正文采用块级结构，块间可插入、删除、移动

#### 5.2.2 标题编辑

- 标题独立编辑
- 支持基础富文本格式
- 标题变化应同步影响文档状态栏与发布内容

#### 5.2.3 块编辑

- 正文由多个块组成
- 每个块支持独立编辑
- 块支持 Enter 分裂、Backspace 合并、拖拽排序
- 空白块应作为自然的插入点

#### 5.2.4 斜杠命令

- 输入 `/` 弹出块插入菜单
- 菜单应支持搜索和快速选择
- 常用块类型应支持最近使用排序
- 插入后保持光标连续编辑

#### 5.2.5 富文本格式

- 支持加粗、斜体、下划线、删除线
- 支持链接插入
- 支持文字颜色与背景高亮
- 支持选区浮动工具条

#### 5.2.6 内容块类型

- 标题：H1 / H2 / H3
- 正文段落
- 引用块
- 无序列表
- 有序列表
- 待办清单
- 分割线
- 表格
- 图片
- 代码块
- 提示框
- 折叠块
- 数学公式
- 嵌入内容

#### 5.2.7 媒体与结构能力

- 支持图片上传、粘贴、拖拽
- 支持封面图设置
- 支持表格创建、增删行列、源码编辑
- 支持公式预览

#### 5.2.8 文档辅助能力

- 支持目录自动生成
- 支持撤销 / 重做
- 支持草稿自动保存
- 支持导入 / 导出 Markdown
- 支持发布前校验

### 5.3 体验要求

- 所有关键操作应尽量在光标附近完成
- 不应频繁打断用户写作流
- 块级操作应保持清晰反馈
- 文档状态需可视化，例如保存中、草稿恢复、字数统计

## 6. 执行流程

### 6.1 用户写作流程

1. 进入写文章页面
2. 输入或粘贴标题
3. 选择封面图（可选）
4. 在正文块中输入内容
5. 使用 `/` 或工具栏插入块
6. 通过快捷键调整格式
7. 必要时插入图片、表格、公式、引用
8. 使用目录检查结构
9. 撤销 / 重做修正内容
10. 预览或复制 Markdown
11. 发布文章

1. 初始化草稿状态
2. 渲染文档级页面
3. 加载标题、块、封面、目录

5. 同步保存草稿
6. 生成导出内容

## 11. 本轮补测结论（2026-06-20）

本轮继续补测了飞书写作页的几个关键交互，补充结论如下：

### 11.1 选区浮动工具条

- 通过程序化选区、双击选词和拖选连续文本尝试触发浮动格式工具条，但当前仍未稳定复现出清晰可识别的工具条面板
- 进一步复测后更准确的口径是：飞书的浮动格式入口更偏向“块级悬停触发”，而不是传统的选中文字后统一弹出
- 工具条按块粒度出现：段落、标题、图片、表格各自对应自己的工具条，换行视为新段落
- 结论：工具条能力应继续保留在需求中，当前仅能标记为“未稳定补测成功”

### 11.2 评论区

- 页面正文底部可稳定看到 `评论（2）`，并且右上区域存在 `跳转至首条评论` 的独立触发入口
- `跳转至首条评论` 不是普通按钮，而是一个带专用类名的评论定位控件，说明评论区具备独立的导航行为
- 紧随其后仍可看到 `评论（2）` 计数展示，说明评论侧栏与正文主视图联动存在

### 11.3 目录与锚点

- 左侧目录项是可点击锚点
- 点击目录项不会改变当前页面 URL，但会在正文内部进行定位/滚动
- 目录与正文之间存在明显的站内联动

### 11.4 图片 / 表格 / 拖拽

- 当前视图里未直接捕捉到标准 HTML `table` 标签，但正文编辑器内部存在块级结构和拖拽句柄，说明表格/图片大概率是编辑器内置块而非原生表格 DOM
- 目前已确认块级拖拽句柄存在，但拖拽排序线和最终重排结果还没有稳定复现成功
- 页面内可见图片入口主要是封面图与头像资源，正文图片/表格仍需要继续在对应块或插入菜单内进一步验证
- 结论：这三类能力仍需继续补测，当前只能保留为“高概率存在，但未完成闭环验证”
- 进一步补充：图片和表格更可能是挂在块级悬停工具条或 `/` 插入菜单中的独立入口，而不是仅靠页面固定工具栏

### 11.5 文档应同步修正的需求口径

- “浮动工具条”应保持为需求项，但不要写成已完全验证
- “图片、表格、拖拽排序”目前更适合写成待补测能力，而不是已确认能力
- “评论跳转首条评论”和“目录锚点联动”已经可以作为确认能力写入结论
- 需求口径应明确区分“选区工具条”与“块级悬停工具条”，避免后续实现误判
